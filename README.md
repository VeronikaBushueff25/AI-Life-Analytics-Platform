Многофункциональная платформа для анализа поведения, личной эффективности и психологической работы над собой с использованием AI.

## Суть проекта

Пользователь регистрируется и каждый день вносит данные: сколько спал, сколько работал, уровень фокуса, настроение и стресс, заметки и тд... Система рассчитывает 5 метрик по научно обоснованным формулам, строит графики динамики и отправляет данные в выбранный AI-провайдер.

**Уникальные инструменты:**
- **AI-профиль личности** — психологический портрет на основе месяцев данных, архетип, корреляционный анализ, прогноз
- **КПТ-практики** — работа с когнитивными искажениями по методу доказательной терапии, ABC-модель с AI-сопровождением
- **Система достижений** — геймификация с confetti-анимацией, 22 уникальных достижения
- **Мультипровайдерный AI** — 4 провайдера

**Целевая аудитория:** студенты, фрилансеры, knowledge workers, люди в периоде выгорания или восстановления.

### Требования

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- [PostgreSQL 15+](https://www.postgresql.org/download/)
- AI-ключ (OpenAI / DeepSeek / HuggingFace / GoogleAi)

### Поддержка HTTP-прокси

Для работы в регионах с ограниченным доступом — настройка прокси в разделе "Настройки":

- Хост + порт
- Логин / пароль (опционально)
- Кнопка "Проверить соединение" — пингует `httpbin.org/ip` и показывает внешний IP

> ⚠️ SOCKS5 (Tor, порт 9150) не поддерживается нативно — используйте Privoxy как HTTP → SOCKS5 мост

## Архитектурные решения

### 1. Clean Architecture

Слои `Domain → Application → Infrastructure → Presentation`. Переход с JSON на PostgreSQL потребовал только замены репозиториев в Infrastructure — Domain и Application не тронуты.

### 2. CQRS + MediatR

Команды (изменение состояния) и запросы (чтение) разделены на уровне классов. Контроллеры содержат только маршрутизацию — логика в хэндлерах. Каждый хэндлер тестируется изолированно.

```
HTTP Request → Controller → IMediator → Handler → Service/Repository → Response
```

### 3. Strategy Pattern для AI

Все провайдеры реализуют `IAIProvider`. `AIProviderFactory` выбирает активный провайдер из DI по настройкам. Контроллеры не знают о конкретных провайдерах (Open/Closed Principle).

### 4. PostgreSQL + EF Core

Миграции версионируют схему и применяются автоматически при старте. Индексы на `(UserId, Date)` для быстрой выборки. Каскадное удаление `User → Activities, Insights, Settings, CbtRecords, Achievements`.

### 5. JWT + BCrypt

JWT — stateless, сервер не хранит сессии. BCrypt с дефолтным cost factor защищает пароли даже при утечке БД. Одинаковое сообщение при неверном email и пароле (защита от перебора).

### 6. Enum вместо строк

Все перечисления — C# enum, сохраняются в БД как строки (`HasConversion<string>()`): читаемость в БД + type-safety в коде.

### 7. Доменное разделение AI-функций

Каждая AI-возможность изолирована в своём сервисе с отдельным промптом: `MetricsService` считает цифры, `PersonalityProfileService` строит психологический профиль, `CbtService` работает с мыслями. Независимое улучшение промптов без влияния на другие функции.

### 8. IHttpClientFactory + кастомный HttpClient для прокси

`IHttpClientFactory` не поддерживает динамическую смену прокси. Для прокси-запросов создаётся `HttpClient` с кастомным `HttpClientHandler` — это осознанное решение, а не антипаттерн.

## Безопасность

- **Пароли** — BCrypt (никогда не хранятся в открытом виде)
- **JWT** — HS256, stateless, срок 30 дней
- **API-ключи** — хранятся в `data/settings.json` или БД, никогда не в коде
- **UserId из токена** — каждый запрос изолирован по пользователю, нельзя получить чужие данные
- **Валидация** — DataAnnotations на всех DTO
- **Прокси** — поддержка HTTP-прокси для закрытых регионов
- **CORS** — настроен (в production ограничить конкретными доменами)

## КПТ-практики

Когнитивно-поведенческая терапия (КПТ) — метод доказательной психологии для работы с деструктивными мыслями.

### ABC-модель (4 шага)

```
Шаг 1 (пользователь): Описать ситуацию и автоматическую мысль
       ↓
Шаг 2 (AI):           Анализ когнитивных искажений + сократовские вопросы
       ↓
Шаг 3 (пользователь): Сформулировать новую, сбалансированную мысль
       ↓
Шаг 4 (AI):           Итоговый поддерживающий комментарий + статистика
```

### Статистика КПТ

- Топ-5 частых искажений пользователя
- Среднее снижение интенсивности эмоции
- Количество завершённых сессий

## AI-профиль личности

Анализирует до 90 дней истории и строит поведенческий психологический портрет.

### Что включает профиль

| Блок | Содержание |
|------|-----------|
| **Архетип** | Уникальное название (напр. "Ночной марафонец 🦉") + описание |
| **Паттерны** | Пик продуктивности, восстановление энергии, триггеры стресса |
| **Суперсилы** | 3 сильных стороны на основе данных |
| **Уязвимости** | 2 ключевых риска |
| **Оптимальные условия** | Идеальный сон, работа, лучший день недели |
| **Корреляции** | Сон↔Фокус, Стресс↔Настроение (коэффициент Пирсона) |
| **Рекомендации** | Топ-3 конкретных действия |
| **Прогноз** | Что будет через месяц + уровень риска (Low/Medium/High) |
| **Полный анализ** | Развёрнутый AI-текст |

### История профилей

Пользователь может видеть как менялся его архетип со временем — достижение "Эволюция" выдаётся при изменении архетипа.
